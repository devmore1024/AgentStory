# AgentStory 页面交互规范

> 当前前台可见品牌统一使用 `AgenTales`，代码与文档文件名暂时保留 `AgentStory`。

## 1. 适用范围

- 一级页面定义为当前主导航入口页：`/`、`/memory`、`/adventure`、`/me`、`/discover`、`/story`
- 次级页面定义为从一级页面继续深入的内容页、详情页、阅读页、进入页、分享页
- 本文只定义页面导航返回交互，不覆盖页面布局、状态文案和业务按钮逻辑

## 2. 返回交互规则

- 所有次级页面必须在主内容区顶部左侧提供统一返回按钮
- 不要只依赖页面底部或操作区中的零散返回入口
- 返回按钮的行为统一为：
  - 有浏览器历史记录时，优先返回上一个页面
  - 无历史记录、用户直接打开链接、或当前环境无法可靠返回时，跳转到该页默认父页面
- 页面内已有的“返回这本童话”“返回同行广场”“返回我的页”等硬编码返回入口，不再作为主返回方案；如需保留，只能作为次要辅助入口

## 3. 默认父页面映射

- `/books/[slug]` -> `/`
- `/books/[slug]/read` -> `/books/[slug]`
- `/books/[slug]/enter` -> `/books/[slug]`
- `/memory/[slug]` -> `/memory`
- `/adventure/[threadId]` -> `/adventure`
- `/me/share` -> `/me`

## 4. 实现约束

- 后续实现必须收口为共享返回组件，不要在每个页面单独拼返回逻辑
- 推荐新增统一组件：`PageBackButton`
- 推荐组件接口：
  - `fallbackHref: string`
  - `label?: string`，默认使用“返回上一页”
- 组件内部统一处理浏览器返回与 fallback 跳转，所有次级页复用同一交互和样式

## 5. 例外

- 一级页面、全局导航页不放返回按钮
- `app/api/*`、`not-found`、错误页不按本规范执行，保留各自独立返回策略
