# AgentStory 动物映射规则

> 面向产品、算法、后端和内容生成链路的首版映射规范
> 更新日期：2026-03-15

---

## 1. 目的

这份文档用于定义 AgentStory 首版如何把 SecondMe 的性格展示结果映射成一个稳定的动物代表。

它解决 5 个问题：

- 为什么用户会得到某个动物
- 系统如何在 8 种动物中做选择
- MBTI 信号如何参与裁决
- 当结果模糊时如何兜底
- 哪些字段需要落到数据和接口里，方便前台解释与后续调优

这份文档默认与 [AgentStory MVP 动物类型范围](/Users/showjoy/devmore/waytoagi/AgentStory/docs/AgentStory%20MVP%20%E5%8A%A8%E7%89%A9%E7%B1%BB%E5%9E%8B%E8%8C%83%E5%9B%B4.md) 配套使用。

---

## 2. 首版映射目标

首版映射不是做“高精度人格测评”，而是做一个：

- 用户看得懂
- 平台讲得清
- 推荐链路用得上
- 分享时有辨识度

的动物人格入口。

因此映射策略优先级如下：

1. 可解释
2. 稳定
3. 差异清晰
4. 推荐可用
5. 精细度

---

## 3. 输入与输出

### 3.1 输入

映射输入来自 SecondMe 性格展示结果，首版统一消费以下文本信号：

- `userInfo`
  - `name`
  - `bio`
  - `selfIntroduction`
- `shades`
  - `shadeName`
  - `shadeDescription`
  - `shadeContent`
  - `shadeDescriptionThirdView`
  - `shadeContentThirdView`
  - `shadeNamePublic`
  - `shadeDescriptionPublic`
  - `shadeContentPublic`
  - `sourceTopics`
  - `sourceTopicsPublic`
- `softMemory`
  - `factObject`
  - `factContent`

如果 SecondMe 原始结构后续变化，映射层仍以“归一化文本”作为主输入，不直接把前台逻辑绑定到某个接口字段。

### 3.2 输出

映射输出包括：

- `animalType`
- `animalName`
- `summary`
- `tendencies`
- `values`
- `expressionStyle`
- `storyPreferences`
- `mappingReason`
- `mappingVersion`
- `confidenceScore`

内部调试字段建议包括：

- `mbtiCandidate`
- `mbtiConfidence`
- `mbtiEvidence`

其中：

- 前台只展示动物人格相关字段
- MBTI 结果只作为内部裁决信号，不直接面向用户展示

## 4. 首版 4 个核心判定维度

为了避免映射过于飘忽，首版仍保留 4 个主维度：

### 4.1 温度

用于判断用户更偏：

- 守护 / 共情 / 陪伴
- 还是冷静 / 观察 / 保持距离

### 4.2 行动力

用于判断用户更偏：

- 主动进入
- 快速决定
- 愿意承担冲突

还是更偏：

- 先观察
- 先感受
- 缓慢进入

### 4.3 思考方式

用于判断用户更偏：

- 理性分析
- 结构判断
- 推演与理解

还是更偏：

- 直觉感受
- 情绪代入
- 氛围感知

### 4.4 表达气质

用于判断用户表达更像：

- 温和安稳
- 轻巧灵动
- 克制清醒
- 锋利直接

## 6. 首版映射方法

首版映射升级为“三段式”：

1. SecondMe 文本信号抽取
2. MBTI 推断层
3. 动物人格综合裁决

这样做的目标不是把产品做成 MBTI 测试，而是让动物人格结果更稳定、更有区分度，同时仍然可解释。

### 6.1 第一层：SecondMe 文本信号抽取

先对 SecondMe 文本做统一归一化，生成一个可回放的文本池。

这层负责：

- 合并 `userInfo / shades / softMemory`
- 对中英文文本统一转小写
- 保留关键词证据，供后续解释使用

### 6.2 第二层：MBTI 推断层

MBTI 层是内部高权重信号，但不直接对前台展示。

它有两类输入方式：

1. 显式 MBTI
   - 文本中直接出现 `INTJ / ENFP / ISFJ` 等字样
2. 语义推断 MBTI
   - 文本里没有明确写出 MBTI，但能从表达、偏好和描述里推断维度倾向

### 6.3 MBTI 维度推断规则

推断 4 组维度：

- `E / I`
- `S / N`
- `T / F`
- `J / P`

每组都采用“正反向信号打分”的方式。

示例方向：

- `E`
  - 社交、表达、互动、带动、外向、热闹
- `I`
  - 独处、观察、内省、安静、克制、深入

- `S`
  - 具体、现实、细节、经验、实际、稳定
- `N`
  - 想象、抽象、可能、联想、隐喻、未来

- `T`
  - 理性、逻辑、分析、判断、客观、拆解
- `F`
  - 共情、温柔、善意、感受、理解、治愈

- `J`
  - 计划、秩序、责任、果断、稳定、收束
- `P`
  - 灵活、探索、即兴、转圜、好奇、开放

裁决规则：

- 若文本中有显式 MBTI，则显式结果优先
- 若没有显式 MBTI，则通过 4 维语义得分生成 `mbtiCandidate`
- 若某些维度信号不足，仍允许得到低置信度 MBTI，但只做轻微影响

### 6.4 第三层：动物人格综合裁决

动物人格裁决不是“MBTI 直接翻译成动物”，而是：

1. 先算动物关键词得分
2. 再叠加 MBTI 对动物的偏置分
3. 最后综合排序，选最高动物
4. 若总体信号不足，则走稳定 hash 兜底

### 6.5 MBTI 到动物的强加权映射

首版不做 16 型逐个硬编码，而是优先看组合倾向：

- `NF` 更容易推向 `deer / rabbit`
- `NT` 更容易推向 `owl / raven`
- `SJ` 更容易推向 `bear`
- `SP` 更容易推向 `cat / fox`
- `NJ` 更容易推向 `owl / wolf`
- `TP` 更容易推向 `fox / raven`
- `FJ` 更容易推向 `deer / bear`
- `TJ` 更容易推向 `wolf / owl`

此外也允许单字母倾向带来较小偏置：

- `E` 对 `fox / wolf`
- `I` 对 `owl / cat / raven`
- `N` 对 `owl / rabbit / raven`
- `S` 对 `bear / cat`
- `T` 对 `owl / raven / wolf`
- `F` 对 `deer / bear / rabbit`
- `J` 对 `bear / wolf / owl`
- `P` 对 `fox / cat / rabbit`

### 6.6 裁决原则

首版采用以下固定裁决原则：

- MBTI 是最高权重信号之一
- 但 MBTI 不能一票否决其他强人格证据
- 显式 MBTI 权重大于语义推断 MBTI
- 高置信度 MBTI 使用强加权
- 中低置信度 MBTI 只做轻微偏置
- 若动物关键词证据非常强，可压过 MBTI 候选
- 低信号场景仍保留稳定 hash 兜底，不回退到统一默认狐狸

## 7. 推荐打分规则

首版建议为每个动物维护一个内部总分，来源分为两部分：

1. 动物关键词分
2. MBTI 偏置分

### 7.1 动物关键词分

继续使用基于关键词和出现次数的规则打分：

- 关键词出现次数越多，分数越高
- 单个高权重词可以快速拉开差异
- 动物关键词负责保留“直接人格证据”

### 7.2 MBTI 偏置分

MBTI 偏置分根据以下因素控制强弱：

- 显式还是语义推断
- `mbtiConfidence` 高低
- 是否与动物关键词一致

建议执行方式：

- 显式 MBTI：强加权
- 高置信度语义 MBTI：中强加权
- 中低置信度语义 MBTI：轻加权

### 7.3 Tie-break 规则

当两个动物得分接近时，按以下优先级打破平局：

1. 优先选择关键词证据更强的动物
2. 再看 MBTI 高权重候选是否明显偏向其中一个动物
3. 优先选择用户更容易理解的动物
4. 优先避免与上一次结果无意义抖动
5. 若仍无法判断，按稳定兜底规则处理

## 9. 低信号与兜底规则

### 9.1 SecondMe 信息不足

当原始结果信息不完整、结构不稳定或标签太少时：

- 先保留当前可识别的动物关键词信号
- 再看是否能形成低置信度 MBTI 候选
- 若整体信号仍不足，则进入稳定 hash 兜底

### 9.2 兜底规则

兜底不应随机，也不应统一回退狐狸。

首版建议：

- 使用用户稳定标识和文本种子做 hash
- 在 8 种动物中做稳定分散
- 同一用户在相同输入下结果保持一致

### 9.3 避免统一默认动物

不再采用：

- 固定回退 `fox`
- 固定回退 `deer`
- 固定回退某个单一“安全动物”

因为这会让低信号用户严重同质化。

## 10. 稳定性规则

### 10.1 避免频繁抖动

如果用户重新生成动物，但新旧结果差异不大，优先保持原动物不变，只更新摘要和偏好。

### 10.2 允许渐进更新

如果用户多次读取 SecondMe 结果后出现明显新信号，可以升级动物类型，但要满足：

- 新动物得分明显高于旧动物
- 变化能被 `mappingReason` 解释
- 推荐结果确实会随之变化

### 10.3 MVP 中不做历史动物展示

首版只保留当前动物，不在前台展示“你曾经是鹿，现在变成狐狸”这类历史轨迹。

## 11. 可解释性输出规范

前台需要能回答用户“为什么我是这个动物”。

因此建议生成一个简短 `mappingReason`，长度控制在 1 到 2 句。

输出原则：

- 优先解释动物结果
- 可以引用命中的人格特征
- 不直接向用户展示 MBTI 编码

示例：

- `系统从你的资料里识别到“观察、分析、秩序感”等特征，因此当前更接近猫头鹰动物人格。`
- `系统从你的资料里识别到“共情、温柔、治愈”这类信号，因此当前更接近鹿动物人格。`
- `你的资料里可直接映射的人格线索还不多，当前先根据已有资料的整体气质为你生成了更稳定的动物人格。`

同时建议保留一个内部 `confidenceScore`，范围可用 `0-100`。

## 12. 数据层建议

为了承接映射解释和后续调优，建议 `AnimalProfile` 或映射结果增加以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| mappingVersion | string | 当前映射规则版本，如 `secondme-v3` |
| mappingReason | string | 前台可展示的映射说明 |
| confidenceScore | integer | 置信度，建议 0-100 |
| mbtiCandidate | string | 内部推断得到的 MBTI 候选 |
| mbtiConfidence | integer | MBTI 置信度 |
| mbtiEvidence | json / array | MBTI 推断使用的证据词 |

`rawSecondMeProfile` 仍保留原始结果，便于后续离线回放和规则升级。

## 13. 接口层建议

`GET /api/me/animal-profile` 建议继续对前台返回：

- `animalType`
- `animalName`
- `summary`
- `mappingReason`
- `confidenceScore`
- `mappingVersion`

内部调试或服务端日志可保留：

- `mbtiCandidate`
- `mbtiConfidence`
- `mbtiEvidence`

其中：

- `mappingReason` 用于人格卡解释区
- `confidenceScore` 首版可不直接前台展示，但建议接口保留
- `mappingVersion` 用于排查线上结果差异
- `MBTI` 默认不暴露给前台页面

---

## 14. 与推荐和生成的关系

映射完成后，不是只得到一个动物名，而是同时得到一组生成偏好：

- `preferredCategories`
- `preferredStyles`
- `preferredRoleTypes`
- `preferredConflictTypes`

因此建议实现顺序是：

1. 先完成动物映射
2. 再生成故事偏好
3. 再驱动首页推荐、连载生成和 AI 自动评论

不要把“动物名”和“推荐偏好”拆成两条互相独立的逻辑。

---

## 15. 首版非目标

以下内容不进入首版映射系统：

- 复杂机器学习模型
- 多动物并存映射
- 面向用户的 MBTI 长报告
- 用户手动修改标签参与判定
- 社区投票决定动物
- 每日波动式人格刷新
- 在前台同时展示“动物人格 + MBTI”

---

## 16. 首版实施建议

首版最推荐的实现方式是：

- 固定 8 种动物
- 固定 4 个主维度
- 固定一套动物关键词规则
- 固定一套 MBTI 维度推断规则
- 固定一套 `MBTI -> 动物` 的强加权映射
- 固定稳定兜底逻辑
- 固定 `mappingVersion = secondme-v3`

这样做的目标不是一次做到最聪明，而是一次做到：

- 对用户足够像
- 对产品足够稳
- 对工程足够好实现
- 对后续调优足够可回放
