# AgentStory 页面级拆解

> 当前前台可见品牌统一使用 `AgenTales`，代码与文档文件名暂时保留 `AgentStory`。

## 1. 首页 `/`

### 页面模块

- Hero：一句话讲清“带着长大后的自己，回到童话里”
- 当前预览卡：展示“你的分身正在回去的童话”
- 童话书架：只保留童话主线，不再前置寓言/神话
- 左下角分身入口：动物人格作为分身的可视化入口

### 模块职责

- 先建立情绪：这不是故事平台，而是重回童话的入口
- 再给选择：挑一本童话，或直接去同行广场看看别人走到了哪里
- 最后承接身份：让用户知道自己的分身会以动物人格出现

### 核心交互顺序

1. 用户进入首页
2. 看见 Hero 和当前回去中的童话
3. 从童话书架里挑一本书
4. 阅读原故事，或直接走进童话
5. 也可以从分身入口进入“我的童话档案馆”

## 2. 分身入口 / 动物人格卡

### 页面模块

- 动物人格主视觉
- 分身名称与人格摘要
- 为什么会是这个动物
- 推荐语气与适合进入的童话
- 分享入口

### 模块职责

- 先解释“这是你的分身”
- 再解释“动物人格只是它出现的样子”
- 给用户一个可以理解、记住、分享自己的方式

## 3. 书详情页 `/books/[slug]`

### 页面模块

- 书封面与基础信息
- 关键情节
- 原故事概览
- 分身提示卡
- `阅读原故事`
- `走进童话` / `继续同行`

### 模块职责

- 让用户在进入前重新理解这本童话
- 说明这次回去会遇见谁、会以什么方式进入
- 区分“先阅读”与“直接走进去”

### 交互规则

- 用户点击 `阅读原故事`：进入阅读页，不立即生成新内容
- 用户点击 `走进童话`：创建一段新的同行故事
- 若用户已拥有该书未完结的故事，则入口改成 `继续同行`

## 4. 阅读页 `/books/[slug]/read`

### 页面模块

- 原故事正文
- 来源信息
- 当前分身会如何进入这本童话
- `回到这本童话`
- `带着分身走进童话`

### 模块职责

- 把阅读原作变成进入前的沉浸过程
- 让用户先重新遇见原故事，再决定如何回去

## 5. 进入页 `/books/[slug]/enter`

### 页面模块

- 进入前 Hero
- 分身身份与进入提示
- 主按钮：`现在走进童话` / `继续同行`
- 次按钮：`去同行广场`

### 模块职责

- 把“生成故事”翻译成“真正走进去”
- 明确这次进入会由哪个分身先开场

## 6. 同行广场 `/adventure`

### 页面模块

- 广场标题与导语
- 我的两项概览：我先走进去的童话 / 我同行过的童话
- 同行卡片列表

### 卡片信息顺序

- 童话名
- 故事标题
- 谁先走了进去
- 最新一段故事
- 当前操作：`继续同行` / `加入同行` / `阅读`
- 次级元信息：人数、篇数、已完结

### 模块职责

- 呈现“与别人的分身同行”这件事
- 机制信息要保留，但只能做次级说明

## 7. 同行详情页 `/adventure/[threadId]`

### 页面模块

- 同行故事头部
- 当前按钮：`继续同行` / `加入同行`
- 章节列表

### 模块职责

- 让用户把注意力放在故事本身，而不是机制推进状态
- 让每一段都像同行中的一次继续靠近

## 8. 我的页 `/me`

### 页面模块

- 动物人格卡
- 我的童话档案馆
- 我的足迹统计
- 同行概览
- 分享资产

### 模块职责

- 呈现“我是谁”
- 呈现“我回去过哪些童话”
- 呈现“我和谁同行过”

### 交互规则

- 时间线统一视为重逢记录 / 同行记录
- 当前页不承担发现职责
- 历史回忆数据继续隐藏，不在这里展示

## 9. 兼容与隐藏入口

- `/memory`：保留提示页，不再进入主产品流
- `/story`：兼容跳转到 `/adventure`
- `/discover`：兼容跳转到 `/adventure`
- 寓言 / 神话旧直链仍可访问，但不再从首页和主导航露出
