# AgentStory 首版风格配置表

> 面向工程接入与内容生成的首版配置表
> 更新日期：2026-03-15

---

## 1. 使用说明

本表用于把 `AgentStory 故事风格规范` 中的风格定义转成可直接接入生成链路的配置项。

建议使用方式：

- 先根据故事分类与内容形态选出候选风格
- 再根据用户动物人格偏好决定最终 `styleKey`
- 根据 `styleKey` 读取对应 prompt 片段和风险提示

## 3. 风格配置项

### 3.1 `fairy`

- 风格名：童话风
- 默认优先级：高
- 推荐使用：
  - 首页自动推荐
  - 连载默认主风格
  - 首次进入某本经典故事

系统提示词片段：

```text
请以童话风写作。语言温柔、具象、带一点神秘感。多使用森林、月光、风、灯火、窗子、斗篷等意象。不要写得幼稚，也不要过于现代口语化。结尾保留一点温暖余韵。
```

生成参数建议：

- 情绪强度：中低
- 节奏：中
- 意象密度：中高

### 3.2 `fable`

- 风格名：寓言风
- 默认优先级：中
- 推荐使用：
  - 寓言类短篇
  - 动物角色主导的内容

系统提示词片段：

```text
请以寓言风写作。结构紧凑、克制、带一点隐喻和判断意味。结尾落在人性观察或选择后果上，但不要说教。
```

生成参数建议：

- 情绪强度：低
- 节奏：快
- 意象密度：中

### 3.3 `epic`

- 风格名：神话史诗风
- 默认优先级：高
- 推荐使用：
  - 神话主线
  - 连载中的重大转折章节

系统提示词片段：

```text
请以神话史诗风写作。语言庄严、命运感强，有神性与仪式感。句子可适度变长，但不要堆砌辞藻，不要写成翻译腔。
```

生成参数建议：

- 情绪强度：中
- 节奏：中慢
- 意象密度：高

### 3.4 `dark`

- 风格名：暗黑风
- 默认优先级：高
- 推荐使用：
  - 危险感和反转更强的短篇
  - 神话惩罚、童话夜路、禁忌主题

系统提示词片段：

```text
请以暗黑风写作。强调不安、压迫、代价和真相的阴影感。对环境细节、身体感受和心理异样保持敏感。允许结尾不圆满，但不要猎奇、血腥或过度审丑。
```

生成参数建议：

- 情绪强度：中高
- 节奏：中慢
- 意象密度：中高

### 3.5 `zhihu`

- 风格名：知乎风
- 默认优先级：高
- 推荐使用：
  - 发现页高传播内容
  - 连载中的复盘型章节
  - 寓言和神话的现代转译

系统提示词片段：

```text
请以知乎风写作。保留故事现场感，同时带有分析、拆因果、讲动机的结构感。读起来像一篇高赞回答里的真实故事，但不要写成纯讲道理。可以使用“先说结论”“后来我才明白”这类结构化叙事习惯。
```

生成参数建议：

- 情绪强度：中
- 节奏：中快
- 意象密度：中

### 3.6 `pain`

- 风格名：伤痛文学风
- 默认优先级：中高
- 推荐使用：
  - 单次高情绪短篇
  - 发现页易传播内容
  - 错过、失去、离别节点

系统提示词片段：

```text
请以伤痛文学风写作。增强主观感受、回望感、遗憾感和错过感。情绪可以浓，但不要堆砌情绪词，不要让内容失去可读性。重点写人物当下的情绪撕扯和事后的回看感。
```

生成参数建议：

- 情绪强度：高
- 节奏：中慢
- 意象密度：中

### 3.7 `light_web`

- 风格名：轻喜剧网感风
- 默认优先级：中
- 推荐使用：
  - 发现页轻量传播内容
  - 轻松短篇
  - 动物角色互动

系统提示词片段：

```text
请以轻喜剧网感风写作。语言轻松、机灵、节奏快，适合分享。可以适度使用现代口语，但不要破坏故事世界的沉浸感，也不要使用过时网络梗。
```

生成参数建议：

- 情绪强度：中低
- 节奏：快
- 意象密度：低

### 3.8 `suspense`

- 风格名：悬疑风
- 默认优先级：中高
- 推荐使用：
  - 连载结尾
  - 暗线推进
  - 章节钩子强化

系统提示词片段：

```text
请以悬疑风写作。通过留白、延迟揭示和问题感推动阅读。结尾保留钩子，但不要故弄玄虚，也不要把关键信息藏得过深。
```

生成参数建议：

- 情绪强度：中
- 节奏：中快
- 意象密度：中

## 4. 连载默认配置建议

### 主风格优先顺序

1. `fairy`
2. `epic`
3. `zhihu`
4. `suspense`

### 连载配置规则

- 一条连载只保留一个主风格
- 单章可允许轻微次风格偏移
- 若章节承担结尾钩子作用，可临时切到 `suspense`
- 不建议让 `pain` 作为长连载主风格
- 不建议让 `light_web` 作为长连载主风格

---

## 5. 短篇默认配置建议

### 主风格优先顺序

1. `fairy`
2. `dark`
3. `pain`
4. `fable`
5. `light_web`

### 短篇配置规则

- 短篇允许更高的风格实验度
- 同一用户可连续阅读不同风格的短篇
- 短篇优先突出一次性风格辨识度

---

## 6. 发现页内容配置建议

### 优先展示风格

1. `zhihu`
2. `light_web`
3. `pain`
4. `suspense`

### 原因

- 更适合分享
- 更容易触发阅读和评论
- 更符合社区内容消费节奏
