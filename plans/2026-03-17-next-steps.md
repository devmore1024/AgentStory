# 接下来要做的三件事

这是当前产品下一阶段的任务清单，按优先级整理，目的是先把影响体验最大的几件事收拢清楚。这里不展开技术实现细节，只明确问题、方向、拆解方式和预期产出，方便后续排期和逐项落地。已确定的产品决策包括：新的两个主 Tab 命名为 `共著` / `我篇`，`发现` 先从导航移除但保留路由与数据逻辑，故事补全优先采用“公有领域原文 + 整理”。

## 1. 首页封面图继续优化

### 当前问题

首页封面已经切到现有封面映射策略和外部图池，但仍然存在部分封面图与故事内容不匹配的问题。当前问题不适合继续靠大批量替换掩盖，应该先把错配原因查清楚，再决定怎么修。

### 要做什么

先做一轮原因排查，不直接进入下一轮全面替换。排查时重点按以下四类归因：

- `slug 精确映射不足`
- `motif 主题池过宽`
- `图像裁切/构图不适合 4:5`
- `素材虽可用但语义不贴故事`

### 为什么先做这个

如果不先定位错配原因，后面无论继续补图、换图还是细化映射，都会反复返工。先把问题分类清楚，后续才能知道是该补精确映射、缩窄主题池，还是只需要调整裁切和选图标准。

### 预期产出

- 一份错配书单
- 一份原因分类表
- 一份按优先级排序的修复清单

## 2. 每个故事补齐可阅读的详细内容与真实关键情节

### 当前问题

现在故事详情页和阅读页主要还是依赖 `summary / originalSynopsis`，可阅读内容仍然偏短。`keyScenes` 里也还有不少是兜底推导文案，不是从完整故事正文里抽出来的真实高光片段，比如“丑小鸭变成天鹅”这种可验证节点还没有被系统化整理。

### 要做什么

把这项任务拆成四步推进：

- `确定公有领域来源`
- `建立故事正文/整理版内容字段`
- `从正文抽取关键情节`
- `把关键情节同步到详情页和阅读页`

任务目标分两层：

- 每本书都应该有可阅读的完整故事版本
- 每本书的关键情节都应该来自故事里的真实精彩片段，而不是泛化兜底文案

### 为什么先做这个

内容层不补起来，首页再好看，用户点进书后也会觉得“没有真正读到故事”。而且关键情节如果不从正文抽取，后续无论是详情页、阅读页还是生成短篇的上下文，都会一直建立在不够扎实的内容基础上。

### 预期产出

- 一份故事来源登记表
- 一份正文整理规则
- 一份关键情节抽取规则
- 一份首批验收样例

## 3. 重构主导航：去掉发现，改为共著 / 我篇

### 当前问题

现在 `发现` 还是单独的主 Tab，而 `连载 / 短篇` 只是故事页内部的子 Tab。这个结构会让“公共共创内容”和“自己的故事内容”都被压在同一个页面里，入口心智不够清楚。

### 要做什么

把 `共著` 和 `我篇` 升级成两个主入口，并明确映射关系：

- `连载 -> 共著`
- `短篇 -> 我篇`

`发现` 的处理方式先固定为：

- 从桌面导航移除
- 从底部导航移除
- 从主要入口文案中移除
- 保留 `/discover` 路由和现有数据逻辑，后续再决定是否并入别处

### 为什么先做这个

当前导航结构先天把内容类型和入口层级混在一起，用户需要先点进“故事”页，才会理解里面还有不同内容。先把主入口改成 `共著 / 我篇`，用户会更容易理解“大家一起写的内容”和“只属于我的故事”是两条不同路径。

### 预期产出

- 一份导航改版清单
- 一份文案替换清单
- 一份保留路由说明

## 建议顺序

1. 先查封面错配原因
2. 再补故事正文和关键情节
3. 最后改主导航与入口命名

这样可以避免先改导航后，内容层仍旧不完整，导致新的入口心智建立起来了，但进去之后还是缺少足够扎实的故事体验。
